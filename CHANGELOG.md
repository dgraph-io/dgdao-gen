# Changelog

All notable changes to this project are documented here. The format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [0.5.1] - 2026-07-16

### Changed

- Bump the `github.com/dgraph-io/dgdao` dependency to v0.6.1, which caches the per-write
  schema check.

## [0.5.0] - 2026-07-10

### Added

- `--license-header <string>` and `--license-header-file <path>` flags that prepend a
  license header, rendered as a valid Go comment, above the
  `// Code generated by dgdao-gen. DO NOT EDIT.` marker in every generated file.
  Input already in comment form (`//` lines or a `/* */` block) is emitted verbatim;
  plain text is wrapped one `//` per line. Omitted flags leave output unchanged.

### Changed

- Bump the `github.com/dgraph-io/dgdao` dependency to v0.6.0.

## [0.4.1] - 2026-07-09

### Changed

- Bump the `github.com/dgraph-io/dgdao` dependency to v0.5.4, which pins Dgraph to the
  released v25.3.8 tag rather than a pre-release pseudo-version.

## [0.4.0] - 2026-07-08

### Changed

- Renamed the module to `github.com/dgraph-io/dgdao-gen` and now depend directly on
  `github.com/dgraph-io/dgdao` (v0.5.3), completing the move off the `mlwelles/modusGraph`
  fork. First version published under the dgraph-io organization.

## [0.3.0] - 2026-06-08

### Added

- Generate `LoadOrStore` and `LoadAndDelete` for upsert-keyed entities.
- `upsertField` template helper for generated code.

### Fixed

- Emit the correct `Code generated by ...` header on generated files.

## [0.2.0] - 2026-06-04

### Changed

- Rename the wrapper-base package `entity` → `wrap`, so generated code imports it
  unaliased (`wrap.Wrapper`, `wrap.WrapValue`). The former `entity` name required an
  `mgentity` import alias to avoid colliding with a consumer's own `entity` package.

## [0.1.1] - 2026-06-04

### Fixed

- Restore the `Models()` aggregate in generated schema-marker files. The initial
  extraction was taken from an early checkout that predated the `Models()` feature, so
  v0.1.0 omitted it; migrate scaffolding and verification (`Provider.Models()`) depend
  on it.

## [0.1.0] - 2026-06-04

### Added

- Initial extraction of the code generator (`cmd/dgdao-gen`,
  `internal/{generator,parser,model}`) and the wrapper-entity runtime (`entity`) from
  dgdao (https://github.com/dgraph-io/dgdao).
- Generated code splits its imports: the generic typed primitives resolve to
  `dgraph-io/dgdao/typed`; the wrapper base resolves to
  `dgraph-io/dgdao-gen/entity`, aliased as `mgentity`.
